# Determinism Contract (Continuum Grid, Stage 0)

## 1. Единицы измерения
- Базовая единица симуляции: `CELL = 1000` fixed units.
- Позиция игрока `x, y` хранится в fixed units.
- Скорость хранится в fixed units per tick.

## 2. Fixed-point
- Тип симуляции: `Fixed = number`, но только целые значения.
- Масштаб: `SCALE = 1000`.
- Вся физика, дистанции и связанные вычисления в reducers выполняются только через fixed-point утилиты.
- Политика округления для fixed-арифметики: **вниз (`Math.floor`)**.
- Проверки:
  - `assertInt` проверяет целочисленность значений (в dev-режиме).
  - `assertSafeRange` контролирует выход за `Number.MAX_SAFE_INTEGER`.

## 3. Сравнение расстояния
- Сравнение радиуса только через квадрат расстояния:
  - `dx*dx + dy*dy <= r*r`
- Запрещено использовать `sqrt` в tick-симации.

## 4. Порядок применения событий
- Входящие команды не меняют мир немедленно.
- Команды складываются в `PendingAction`.
- На каждом тике применяется сортировка и обработка в порядке:
  - `(tick, playerId, seq)`
- Порядок должен быть тотальным и воспроизводимым.

## 5. PRNG
- Разрешён только детерминированный PRNG от seed.
- Запрещён `Math.random()` в симуляции.
- Seed фиксируется как 32-битное значение.
- `nextFloat01()` допускается для процедурной генерации мира.
- `nextFloat01()` не используется в боевой тик-симуляции.

## 6. Tie-break правила
- Любой выбор "лучшего" (минимум/максимум стоимости, путь, цель и т.п.) обязан иметь tie-break.
- При равенстве выбирается меньший `id`.

## 7. Запреты (источники недетерминизма)
- `Math.random()`.
- `Date.now()`, `performance.now()`, время ОС и любые wall-clock источники.
- Неупорядоченный обход структур без сортировки (например, `Map`, `Set`, `Object` ключи) в симуляции.
- Использование float-логики в reducers вместо fixed-point.
- Алгоритмы выбора без явно заданного tie-break.
